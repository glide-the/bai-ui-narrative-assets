# 工程设计稿

## 1. 背景与目标
- 本文是 SUO-68 设计阶段的权威工程规格，承接 `docs/architecture/项目架构设计说明.md` 所描述的三层文档体系，参考 `claude-agent-next-kit` 同类章节结构，确保设计与架构层面同向。
- 目标是明确叙事型 UI 的体验、资产管线、交互与数据方式，为下游 Agent 留下可执行的设计真相源，避免重复生成多个版本。
- 该设计稿负责叙事内容与浏览器原型工程的规格；具体的运行、构建、CI 与后端实现由 `docs/architecture` 与 `docs/solution` 接手。

## 2. 范围界定
- 仅在 `docs/design/` 维度记录叙事文本、符号资产、交互细节、数据结构与验收标准；不写 Issue / Task / Stage 文档，也不记录执行命令或运行策略。
- 原型聚焦单屏、横向故事板，场景与符号均由项目内部生成的资产包完成；不在本文描述 Claude Agent API 或其它运行时实现。
- 全局变量、色彩、尺寸、文案、路径等常量放在 config / data 模块统一管理；设计稿与实际代码均应避免散落硬编码值。

## 3. 方案摘要
- 以 1440×900px 的桌面视图承载七章故事：顶栏 + 叙事面板 + 场景画布 + 符号热区 + 底部章节轨。
- 在正式渲染前通过脚本生成 `/assets` 中的 SVG/PNG，生成即为消费：资产清单、manifest、符号卡片由同一管线产出。
- 数据驱动章节与符号（config、章节数据、符号 manifest），配合全局 token（颜色/尺寸/文本）实现一致视觉与交互体验；此结构参照 `claude-agent-next-kit/docs/architecture/...` 的工程 / 模块 / 目录分层。

## 4. 详细设计
### 4.1 体验方向与叙事结构
- 以 1440×900px 桌面首选视口承载七章横向故事：顶栏、左叙事、中央/右场景、符号热点与章节轨组成五区布局，主画面不产生纵向滚动，所有信息集中在一屏。
- 七章按古道、村寨、本主、三道茶、花与日常、民居之困与新的共生展开，每章包含章节标题、副标题、关键词、场景、符号集合、现代转折笔记。
- 交互由章节按钮、符号热区、现代转折叠层、符号图谱、拖拽滑块、键盘快捷键（←/→/1-7/M/A/G/R/S）与 localStorage 记忆协调完成，保持故事节奏与可访问性。
- 视觉风格混合文化展陈与现代界面：米色纸张、石纹/布纹质感、黑色剪影、黏土红与靛蓝高亮、屏风卡片与线性进度，避免廉价民族装饰与工具化仪式堆砌。
- 符号体系覆盖身份/生活/仪式三类 A/B/C 资产，每个符号在 manifest 中含 narrativeFunction、chapter 关联与 assetPath，供叙事、图谱与字典互用，避免组件中硬编码文案。
- 本体验定位为可直接打开 `index.html` 的浏览器原型，所有可见内容均源自本仓库生成的 `/assets` 资源或代码绘制元素，禁止引用外部素材、图标库或占位图；详情见第 4.2 节与 `docs/architecture/项目架构设计说明.md` 的运行描述。
- 运行、构建与后端交付脱离本文件，由 `docs/architecture` 与 `docs/solution` 分工，本设计稿聚焦体验、资产与交互规范。

### 4.4 章节叙事内容
Narrative chapters:

Chapter 01:
Title:
古道：文化从流动中来

Narrative goal:
Show that Bai culture is formed through movement, trade, exchange, ancient roads, migration, and regional openness.

Scene:
Ancient road line crossing the page like a path and Erhai waterline.
Use bird, clay pot, and sun spiral as visual symbols.

Symbols:
B6 bird
B7 clay pot
C3 sun spiral

Main copy:
大理不是孤立的地方。
古道经过这里，商旅、信仰、器物、语言和风俗在这里交汇。
文化不是静止的遗产，而是一次次经过、停留、交换之后留下的生活纹路。

Modern transition:
今天的高速路、机场、手机地图和文旅路线，延续了古道的功能。
只是流动的形式变了：过去是马帮、器物和口述传说，今天是游客、影像、品牌和数字信息。

Chapter 02:
Title:
村寨：身份被看见

Narrative goal:
Show how identity becomes visible through headwear, earrings, collars, garments, and body decoration.

Scene:

Symbols:
A1 feather crown / headwear
A3 round earrings
A5 collar / neck ornament
A8 crown ornament

Main copy:
身份首先不是一句介绍，而是被看见的方式。
头饰、耳饰、项圈、衣边，让一个人从普通身体进入族群秩序。
在村寨里，身份不是孤立的个人标签，而是人与家庭、节庆、礼俗之间的关系。

Modern transition:
今天，身份被更多界面重新编码。
头像、昵称、证件、品牌、社交媒体和城市职业，构成新的身份系统。
但在白族文化里，身体装饰曾经就是最直接的界面。

当现代生活进入村寨，仪式没有消失。
它改变了功能：从单纯祭祀，转向聚会、表演、交流和共同体记忆。

## 5. 验收标准
- 以 1440×900 框架承载七章横向故事与底部轨道（01~07），主体验整合：顶栏、叙事面板、场景画布、符号热区；主画面不得产生纵向滚动。
- 所有视觉内容来源于本项目生成的 `/assets` 资产包，生成脚本、asset manifest、符号目录、运行说明（打开 `index.html` 即可）与代码文件一并提供；不得依赖外部素材或占位图。
- 交互必须支持：章节按钮、符号热点、过渡悬浮提示、现代转折叠层、比对滑块、符号图谱弹窗、键盘导航（←/→/1-7/M/A/G/R/S）、布局指引开关、历史位置存储与重置。
- 可访问性要求：焦点态、ARIA 标签、色彩对比、非色彩状态提示、键盘可达、现代转折/图谱的辅助说明。
- 项目必须清楚说明如何复现：文件树、运行步骤、asset 生成方式、怎么验证所有符号链接和章节数据都被包含（见 deliverables 6 条的说明版本化记录）。

## 6. 风险与依赖
- 资产生成管线缺失或与 manifest 不一致将直接影响原型可交付性，依赖严格按 `assets` 结构生产的符号与纹理文件。
- 章节数据、符号 manifest、CSV / Markdown 表格需与 UI 设计稿同步，否则交互侧无法自动驱动；数据更新必须同步更新本文件与 `docs/design/民族图案_*` 数据源。
- 仓库根目录已含 Next.js（Claude Agent API）；若叙事原型与运行时职责再分工，须在 `docs/architecture` 以及 `docs/solution` 中更新边界，避免本设计稿与运行时描述冲突。
- 交互与可访问要求依赖浏览器原生能力（无后端），需要在 `index.html` / `/src` 的架构中清晰体现控制流程与状态管理。

## 7. 关键决策记录
- 以五区布局（Top header、Left narrative、Center/right canvas、Bottom rail、symbol popovers）承载七章故事，保持单屏无纵向滚动的视觉连贯。
- 採用数据驱动章节与符号，所有全局值（token、路径、键盘快捷键）集中在 config 模块，防止硬编码全局变量。
- 生成式资产（SVG/PNG/Canvas）必须先产出再引用，并生成清晰的 manifest 及 metadata，重复利用布局与符号中的每个元素。
- 交互集包含章节切换、热点、现代转折、符号图谱、拖拽 slider、键盘与 localStorage 助手，以保证故事沉浸与可访问性。
- 文档结构参照 `claude-agent-next-kit/docs/architecture/项目架构设计说明.md` 的工程 / 模块 / 目录划分，保持与 `docs/architecture`、`docs/solution` 之间的清晰边界。
