# MVP 验收清单（对照设计资产）

> 依据 `docs/design/任务说明.md` 与 `UI设计资产说明.md` 抽取。**未写死技术栈**：适用于首版可交互原型或展陈端。

验收时建议每条对应：演示步骤 + 截图或录屏 + 已知缺陷备注。

---

## A. 叙事与信息架构

| # | 验收项 | 来源 |
| --- | --- | --- |
| A1 | 用户可在界面中识别全部 **7 个章节**，顺序与 `任务说明` 一致 | 任务说明 §3.2 |
| A2 | 每章呈现「传统 → 现代」转折语义，不与 `完整叙事设计资产` 明显冲突 | 任务说明、完整叙事 |
| A3 | 符号以「身份 / 生活 / 仪式」角色参与叙事，而非孤立图鉴 | 任务说明、UI 说明 |

---

## B. 单屏与导航（核心体验）

| # | 验收项 | 来源 |
| --- | --- | --- |
| B1 | **首屏内**完成主要叙事布局：Header + 双栏 + Chapter Rail，无依赖纵向整页滚动 | UI 说明 §2、§4 |
| B2 | 章节切换通过 **底部横向 Chapter Rail**（或等效、可访问的横向控件）完成 | 任务说明 §3.3 |
| B3 | 在目标画布 **1440×900** 下无重叠遮挡类布局错误；在 **1180×820**、**min 1024 宽** 下有关键信息不裁切策略 | UI 说明 §3 |

## C. 交互（与说明一致的最小集）

| # | 验收项 | 来源 |
| --- | --- | --- |
| C1 | 可切换章节并更新：标题区当前章节、旁白区内容、场景区主视觉 | 任务说明 §3.3 |

## E. 非目标（本 MVP 可不验）

* 登录、账号、后端持久化会话。
* 多语言完整翻译（除非单独立项）。
* 商业级性能压测（可在 `docs/solution` 另开性能里程碑）。
