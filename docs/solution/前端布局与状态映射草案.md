# 前端布局与状态映射（草案）

> 依据 `docs/design/UI设计资产说明.md` 的页面分区，给出 **实现侧** 的组件边界与状态机草案。未锁定框架版本前，以「职责 + 数据依赖」为主，便于 React / Vue / 原生展陈任选其一落地。

---

## 1. 布局 → 组件（建议命名）

| UI 区域（设计规格） | 建议组件 / 模块 | 主要职责 |
| --- | --- | --- |
| Header（~88px 高） | `StoryHeader` | 主副标题、当前章节名、横向进度、全局样式 token |
| Narrative 左栏（~360px 宽） | `NarrativeColumn` | 章节编号、标题、旁白、现代转折、关键词标签 |
| Scene Canvas 右区 | `SceneCanvas` | 场景合成层：背景、符号热点、局部点击区 |
| Detail Panel / Symbol Tooltip | `SymbolDetailSheet` / `SymbolTooltip` | 符号详情；弹层与轻提示二选一或组合（以 UI 说明为准） |
| Chapter Rail 底部 | `ChapterRail` | 七章切换、当前高亮、键盘/触摸可达性 |

可选横切模块：

* `LayoutShell`：画布尺寸断点（1440×900 / 1180×820 / min 1024 宽）与禁止纵向滚动的根约束。
* `ProgressStrip`：若与 Header 中的进度线拆分，可独立为子组件。

---

## 2. 最小应用状态（概念）

| 状态 | 类型（概念） | 说明 |
| --- | --- | --- |
| `activeChapterIndex` | `0..6`（七章） | 与 `任务说明` 七章顺序一致；驱动旁白、场景资源、Rail 高亮 |
| `selectedSymbolId` | `string \| null` | 来自符号表主键或稳定 slug；驱动 Tooltip / Detail |
| `detailVisibility` | `enum` | `closed` / `tooltip` / `panel`（具体枚举与 UI 说明中的交互一致即可） |
| `locale` | 可选 | 若未来双语，再接入；当前资产以中文为主 |

**派生数据**：当前章节的叙事段落、关键词列表、场景引用图 —— 建议从结构化内容层读取（见下节），组件不硬编码长文案。

---

## 3. 内容与设计数据管线（建议）

| 资产 | 路径 | 实现侧用法 |
| --- | --- | --- |
| 七章叙事与章节元数据 | `docs/design/任务说明.md`、`完整叙事设计资产.md` | **MVP**：构建脚本或手工 JSON 切片；**演进**：MDX / CMS |
| 符号几何与语义 | `民族图案_身份生活仪式符号表.csv`、`民族图案_身份生活仪式符号表_mask坐标系.md` | 构建时解析 CSV → 运行时只读列表；mask 坐标用于 Canvas 热点或多边形命中 |
| 视觉常量（色、间距、字号） | `UI设计资产说明.md` | 提炼为 design tokens（CSS 变量或主题对象），避免在组件内魔法数 |

原则：**设计文档仍为单一事实来源**；代码仓内的 JSON / 常量文件若是派生产物，须在 `docs/solution` 或构建 README 中注明生成方式。

---

## 4. 与 Figma 的对接（若采用）

* 每个 `UI设计资产说明` 中的「区域」应对应 Figma 中命名一致的 Frame（便于 diff 与 handoff）。
* 导出切图与矢量策略：静态背景与可替换纹理走资源管道；符号若向量化，保持与 CSV 中 id 一致。

---

## 5. 未决项（留给立项后 ADR）

* 是否引入路由（多 URL 章节）还是纯客户端状态机单页。
* 展陈全屏浏览器 vs 嵌入式 WebView 的输入事件差异。
* 无障碍：章节切换与符号聚焦顺序的 WCAG 目标级别。
