# docs/solution — 落地方案

本目录用于存放 **从设计资产到实现** 的方案类文档，与 `docs/design`（体验与内容规格）和 `docs/architecture`（仓库与信息架构边界）分工一致，对齐 `claude-agent-next-kit` 的三层文档划分。

## 建议放置内容

| 类型 | 说明 |
| --- | --- |
| 技术选型备忘录 | 例如 React / Vue / 原生展陈播放器、是否引入 CMS |
| 组件与状态映射 | 将 `UI设计资产说明` 中的区域映射到可复用组件与状态机 |
| 数据管线 | CSV / Markdown 如何进入构建时数据或运行时配置 |
| 验收清单 | 与七章叙事、单屏无纵向滚动等目标的对照表 |

## 与相邻目录的关系

```text
docs/architecture  →  为什么这样分目录、仓库边界
docs/design        →  用户看到什么、文案与交互规格
docs/solution      →  怎么实现、谁对接、如何验收
```
