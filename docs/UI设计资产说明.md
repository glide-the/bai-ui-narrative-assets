# UI设计资产说明

## 1. 设计定位

项目名称：**洱海边的三重回声**

副标题：**从白族文化到现代文明的视觉叙事**

设计类型：**单屏横向故事板 UI / 民族文化数字叙事界面 / 文旅视觉交互系统**

本 UI 不是常规符号库，也不是静态图录，而是一个通过章节切换推进的文化叙事界面。页面将“身份符号、生活符号、仪式符号”转化为故事节点中的视觉角色，使用户在一屏内理解白族文化如何从传统生活进入现代文明。

---

## 2. 核心 UI 目标

1. 在一屏内完成叙事组织，避免长屏滚动。
2. 让符号成为故事的一部分，而不是孤立装饰。
3. 在传统材料感与现代 UI 组件之间建立统一视觉语言。
4. 通过横向章节切换表现时间、文化流动与现代转化。
5. 为后续 Figma、React、展陈屏或网页首屏开发提供清晰结构。

---

## 3. 推荐画布尺寸

```text
Desktop / 展陈横屏：1440 × 900 px
Tablet 横屏：1180 × 820 px
最低适配宽度：1024 px
```

主体布局以 `1440 × 900 px` 为基准设计。  
所有主要信息必须在首屏范围内完成呈现。

## 5. 页面区域说明

### 5.1 Header 顶部标题区

高度：`88 px`

内容：

- 主标题：洱海边的三重回声
- 副标题：白族文化与现代文明叙事界面
- 当前章节名称
- 横向进度线

布局建议：

```text
左侧：项目标题
中间：章节标题
右侧：章节进度 03 / 07
```

视觉处理：

- 底部使用 2px 墨黑分割线
- 进度线使用陶土红
- 标题区背景使用米纸色
- 可加入非常浅的纸纹噪点

---

### 5.2 Narrative 左侧叙事旁白区

宽度：`360 px`

作用：

- 承载章节标题
- 展示章节旁白
- 显示本章核心问题
- 提供“传统 → 现代”的转折说明

结构：

```text
章节编号
章节标题
一句话主题
叙事旁白
现代转折
关键词标签
```

示例：

```text
03
本主：神圣走近人间

本主不是遥远的神。
它在村头，在庙里，也在人们的日常判断里。

现代转折：
仪式逐渐从单纯娱神，转向聚会、表演、交流和共同体记忆。

关键词：
本主 / 村寨 / 共处 / 仪式 / 公共文化
```

视觉建议：

- 左侧旁白区保持安静，不使用复杂图案
- 使用大留白
- 关键词以小胶囊标签呈现
- 标题使用黑色，现代转折使用陶土红强调

---

### 5.3 Scene Canvas 右侧场景画面区

宽度：`约 980 px`  
高度：`约 660 px`

作用：

- 呈现当前章节的视觉场景
- 放置本章所需符号
- 承载局部交互、弹层入口、动态转场

视觉结构：

```text
背景层：米纸 / 石纹 / 布纹 / 白墙青瓦意象
场景层：古道线、村寨轮廓、本主庙、茶杯、花枝、民居对比等
符号层：A/B/C 符号图案
交互层：热点、Tooltip、选中描边、详情卡
```

符号展示原则：

1. 每章只显示 3 到 7 个核心符号。
2. 不同时显示全部 25 个符号。
3. 符号大小根据叙事层级变化。
4. 主符号最大，辅助符号较小。
5. 被点击符号可进入详情卡或弹层。

---

### 5.4 Detail Panel 右侧局部详情卡

详情卡可作为 Scene Canvas 内部浮层出现，不单独占用固定右栏。

触发方式：

- 点击符号
- Hover 符号
- 点击章节关键词
- 点击场景热点

内容结构：

```text
符号编号
符号名称
所属类别
叙事功能
转喻说明
适用场景
```

示例：

```text
A1 羽冠 / 头饰
类别：身份符号

叙事功能：
用于表现“身份被看见”。

转喻说明：
头饰不是单纯装饰，而是身体进入族群秩序时最先被看见的标记。

适用场景：
村寨、节庆、人物身份、仪式开场。
```

视觉样式：

- 背景：骨白色
- 边框：2px 墨黑
- 阴影：手工版画式硬阴影
- 选中强调：陶土红角标

---

## 7. 色彩系统

```css
:root {
  --ink-black: #11100D;
  --rice-paper: #F4EEDF;
  --bone-white: #FFF8E8;
  --warm-sand: #D8B982;
  --clay-red: #9B3F2F;
  --indigo: #263C50;
  --stone-gray: #9A968A;
  --leaf-green: #556B46;
  --shadow-brown: #4A3428;
}
```

### 7.1 主色

- `--rice-paper`：页面底色
- `--ink-black`：符号、标题、边框
- `--clay-red`：当前状态、重点、仪式感

### 7.2 辅助色

- `--warm-sand`：旧纸、卡片底纹
- `--indigo`：现代信息层、英文副标题
- `--stone-gray`：民居、石墙、次级文字
- `--leaf-green`：花与日常章节辅助色

### 7.3 使用原则

1. 黑色用于符号主体，不使用彩色填充破坏剪影感。
2. 陶土红只用于强调，不大面积铺底。
3. 米纸色贯穿全局，统一传统纸本气质。
4. 现代 UI 组件可以使用靛蓝，但比例要低。
5. 不使用荧光色、霓虹色、廉价渐变。

## 12. 交互说明

### 12.1 横向章节切换

用户通过底部章节导航切换故事节点。  
章节切换时，画面不纵向滚动，而是横向滑入或淡入。

推荐转场：

```text
古道 → 村寨：古道线延展成村寨轮廓
村寨 → 本主：头饰纹样变成本主庙门框
本主 → 三道茶：太阳旋涡缩小成茶杯蒸汽
三道茶 → 花与日常：茶气变成花枝
花与日常 → 民居之困：花枝退后，露出民居墙面
民居之困 → 新的共生：建筑碎片重组为 UI 卡片
```

### 12.2 符号点击

点击符号后展示详情卡。  
详情卡不跳页，不打断叙事。

### 12.3 场景热点

某些场景元素可点击：

- 古道线
- 本主庙门
- 茶杯
- 花枝
- 民居滑块
- 现代 UI 卡片

### 12.4 民居之困滑块

第 06 章提供传统与现代对比滑块。

```text
传统民居 ←→ 现代楼房
```

滑块左右移动时切换：

| 传统 | 现代 |
|---|---|
| 天井 | 客厅 |
| 照壁 | 外立面 |
| 石墙 | 水泥墙 |
| 院落 | 单元房 |
| 手工装饰 | 工业材料 |

## 13. 不做长屏滚动的解决方案

### 13.1 使用章节切换代替纵向滚动

7 个章节通过底部导航切换，每章占一个屏幕状态。

### 13.2 使用弹层承载详情

符号说明、坐标、文化解释、前端资源信息全部放入弹层或 Popover。

### 13.3 使用横向进度线承载时间

文化演变不是用纵向页面表示，而是通过横向时间线表示。

### 13.4 使用局部切换承载对比

现代与传统的冲突通过滑块、Tab、淡入淡出表现，不通过上下对照长页面表现。

### 13.5 使用符号缩略索引承载全量资产

全部 A/B/C 符号不在主页面一次铺开。  
需要时可从“进入符号图谱”打开独立符号库页面。

## 14. 符号使用规范

### 14.1 符号作为叙事角色

每个符号必须承担至少一个叙事功能：

```text
身份识别
生活说明
仪式连接
现代转化
冲突表达
文化收束
```

### 14.2 符号尺寸

```text
主符号：120–180 px
辅助符号：64–96 px
背景水印：220–360 px，透明度 4%–8%
详情弹层符号：96–140 px
```

### 14.3 符号颜色

默认使用墨黑色。  
在选中或章节强调时可以使用陶土红描边，不建议直接改为红色实体。

### 14.4 符号命名

资源文件命名建议：

```text
a1-headdress.svg
a2-profile-headwear.svg
a3-round-earrings.svg
b1-mortar-pestle.svg
b7-clay-pot.svg
c3-sun-spiral.svg
c8-sacred-bird.svg
```

### 14.5 Mask 使用

如果从原图 bbox 切图，每个 bbox 外建议增加 `8–16 px` padding，避免边缘裁切。  
前端不要直接使用整张长图定位展示，应拆分为独立 SVG 或 PNG mask 资产。

## 15. 动效建议

### 15.1 基础动效

```text
章节切换：360ms ease-out
符号 hover：180ms ease
详情卡出现：220ms ease
背景纹理淡入：480ms ease
底部进度移动：300ms ease
```

### 15.2 章节动效

| 章节 | 动效建议 |
|---|---|
| 古道 | 曲线缓慢绘制 |
| 村寨 | 身份符号逐个拼合成人物轮廓 |
| 本主 | 太阳旋涡缓慢转动，手掌符号亮起 |
| 三道茶 | 三只茶杯依次浮现 |
| 花与日常 | 花枝从左下角生长 |
| 民居之困 | 滑块对比传统与现代 |
| 新的共生 | 传统符号重组为现代 UI 卡片 |

### 15.3 动效限制

- 不使用强烈弹跳
- 不使用霓虹光效
- 不使用过度 3D 翻转
- 不让动效盖过文化内容

## 16. 前端实现建议

### 16.1 推荐技术栈

```text
React
Tailwind CSS
Framer Motion
SVG Mask
CSS Variables
```

### 16.2 页面组件树

```text
NarrativeAtlasPage
├── HeaderBar
├── MainStage
│   ├── NarrativePanel
│   └── SceneCanvas
│       ├── TextureBackground
│       ├── ChapterScene
│       ├── SymbolHotspot[]
│       └── SymbolDetailPopover
└── ChapterRail
```

### 16.3 数据结构

章节数据与符号数据应分离：

```text
chapters.ts
symbols.ts
```

章节引用符号 ID，不直接写死符号内容。

示例：

```js
{
  id: "chapter-03",
  title: "本主：神圣走近人间",
  symbols: ["C1", "C2", "C3", "C4", "C6"],
  modernShift: "从祭祀场所转向公共文化活动",
  mood: ["庄重", "亲近", "共同体"]
}
```

### 16.4 响应式策略

桌面端保持左右结构。  
平板端左侧旁白可压缩为顶部卡片。  
移动端如果必须适配，可变为章节卡片式，但此版本优先保证横屏体验。

---

## 17. 视觉禁忌

1. 不要把所有民族符号铺满页面。
2. 不要使用大红大绿的廉价民俗配色。
3. 不要把传统文化处理成旅游海报。
4. 不要用过多复杂花纹降低信息可读性。
5. 不要让页面变成百科长文。
6. 不要把现代文明简单处理成“破坏者”。
7. 不要把传统简单处理成“静态遗产”。

---

## 18. UI 一句话方向

> 用现代界面的秩序，承载白族文化的身份、生活与仪式；用单屏横向故事板，让传统不再只是被观看，而是重新进入现代生活的表达系统。
