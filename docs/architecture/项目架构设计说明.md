## 仓库定位与边界

| 层级 | 职责 | 非职责（本阶段） |
| --- | --- | --- |
| **资产层** | 可版本化正文与表格数据（叙事、规格、符号表） | 不托管生产密钥、不内嵌可执行后端 |
| **文档层** | 三类目录分工（architecture / design / solution）与交叉引用规范 | 不替代外部设计主交付物（如 Figma）的权威源 |
| **演进层** | 后续可增 PoC、Storybook、子包等；须在本文档更新边界 | 未立项前不在本文假设具体框架版本 |

# 架构设计

### 公共模块

> 类比上游「公共模块」表：此处每条对应 **一类文档资产路径域**；「通用能力」体现在跨目录命名、交叉引用与演进约束（由本文与 ADR 承载），而非 `app/lib` 代码包。

| 模块（路径域） | 用途 |
| --- | --- |
| `docs/architecture/*.md` | 仓库结构、边界、与外部系统关系；架构事实单一登记点 |
| `docs/design-system.md` | Tokens、组件规范、符号命名、章节与交互状态（跨 design / solution 对齐用） |
| `docs/design/工程设计稿.md` | 单屏原型 **目录/交互/资产生成/技术约束** 的工程规格（实现权威） |
| `docs/design/任务说明.md` | 项目目标、章节结构、受众（规格入口，非本文展开） |
| `docs/design/UI设计资产说明.md` | 布局分区、组件行为、文案层级 |
| `docs/design/完整叙事设计资产.md` | 连续叙事正文交付物 |
| `docs/design/*.csv`、`docs/design/民族图案_*_mask坐标系.md` | 结构化符号与坐标 / 蒙版约定 |
| `docs/solution/*.md` | 实现路径、对接、验收与管线说明 |
| `docs/README.md` | 文档中心入口：背景摘要与资源索引（职责外于本文） |

### 基础业务模块说明

> 类比上游「页面 + API + lib」闭环：本仓以 **规格资产（design）→ 落地方案（solution）→ 结构登记（architecture）** 形成文档闭环。

**核心文档业务域**：叙事与任务、UI 规格、符号数据、原型与资产管线。

| 文档业务域 | 规格与设计资产（`docs/design`） | 实现与对接（`docs/solution`） | 结构与边界（`docs/architecture`） |
| --- | --- | --- | --- |
| 叙事与任务 | `任务说明.md`、`完整叙事设计资产.md` | `MVP验收清单.md`（叙事对照项） | 三层目录边界、本文「业务架构模块」树 |
| UI 规格 | `UI设计资产说明.md` | `前端布局与状态映射草案.md` | 同上 |
| 符号数据 | `*.csv`、`*_mask坐标系.md` | 管线与组件数据入口（见 solution 草案） | 同上 |
| 原型与资产管线 | `工程设计稿.md`、`design-system.md`（规范） | `前端布局与状态映射草案.md`、`README.md` | 根 `README.md`、演进层、本树 |

### 约束与自动化扩展位

> 类比上游「认证中间件模块」：**当前未引入** 登录鉴权、CI 强制闸门或 schema 单测；资产默认面向受控协作环境（PR / 评审）。

若后续需要接入自动化或强约束，建议预留落点：

* **根目录或 `.github/workflows/*`**：PR 校验、链接检查、basename 规范
* **`docs/architecture/ADR-*.md`**：记录选型与不可逆约束
* **可选 `scripts/` 或 `package.json`**：数据校验、CSV schema 检查（立项后纳入树与本文）

---

### 资产交付与协作上下文

> 类比上游「Step 消息 / Handler Context」：本仓无运行时 Handler，以下为 **人机协作与变更追溯** 的约定字段位。

| 属性（约定） | 功能用途 |
| --- | --- |
| 变更载体 | PR / Issue 编号；大段叙事改动建议附对比摘要 |
| 目录职责 | 变更须落在 design / solution / architecture 之一，避免跨层重复事实 |
| 回滚语义 | Git revert；ADR `superseded` 链维护架构事实 |
| 外部权威 | Figma / CMS 等若引入，须在 `docs/solution` 登记对接面，由本文「演进层」更新边界 |

## 与参考项目的差异（避免误用）

* **claude-agent-next-kit**：Next.js App Router + `app/lib` + Route Handlers 的 **单体 Web 应用** 架构说明；含运行时、DB、AI SDK 等模块表。
* **本仓库**：**文档与数据资产优先**；上表各「模块」均为 **文档路径域**。若未来增加应用代码，应在本树增加 `app/`（或等价）并新增子章节登记，与上游形态收敛。
